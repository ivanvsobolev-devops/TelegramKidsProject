# Development Process

## Methodology

Spec Driven Development

## Feature Workflow

1. Understand request.
2. Analyze architecture impact.
3. Create specification.
4. Create ADR if necessary.
5. Review assumptions.
6. Produce implementation plan.
7. Implement.
8. Test.
9. Update documentation.

## Specification Template

Each specification must contain:

- Problem
- Scope
- Out of Scope
- User Stories
- Functional Requirements
- Non-functional Requirements
- Acceptance Criteria

## Definition of Done

Feature is complete only when:

- Code implemented
- Tests added
- Documentation updated
- CI passing
- Acceptance criteria satisfied
